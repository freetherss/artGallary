### 1. 프로젝트 개요
- **목표:** 아티스트가 자신의 작품을 손쉽게 온라인에 게시하고 관리할 수 있는 개인화된 웹 갤러리 공간을 제공합니다.
- **핵심:** 직관적인 업로드 및 관리, 작품에 집중할 수 있는 깔끔한 UI, 해시태그 기반의 간단한 필터링 기능을 구현합니다.
- **참고 문서:** `docs/PRD.txt`, `docs/SRS.txt`

### 2. 개발 환경 점검
- **요구사항:** Java (JDK), Gradle, MySQL, Node.js, Git.
- **구현 상황:** 모든 필수 소프트웨어의 설치가 완료되었습니다.
- **남은 작업:**
    - MySQL에 `jiw-gallery` 데이터베이스 생성.
    - 프로젝트에서 사용할 사용자 계정 및 권한 설정.

### 3. 1단계: 백엔드 기초 및 데이터 모델링
- **요구사항:** 게시물(이미지, 설명글, 해시태그) 정보를 데이터베이스에 저장할 수 있어야 합니다.
- **구현 방안:**
    - **JPA Entity 정의:** `Post`라는 이름의 JPA Entity를 생성합니다. 필드는 `id` (Long), `description` (String), `imagePath` (String), `hashtags` (String) 등을 포함합니다.
    - **데이터베이스 연결:** `src/main/resources/application.properties` 파일에 MySQL 접속 정보를 설정합니다 (URL, username, password).
    - **초기 스키마:** Spring Boot 실행 시 JPA가 `posts` 테이블을 자동으로 생성하도록 설정합니다.

### 4. 2단계: 핵심 API 개발 (게시물 CRUD)
- **요구사항:** (SRS: FR-1, FR-2, FR-5, FR-6) 이미지 업로드, 게시물 생성/수정/삭제 기능을 제공하는 API가 필요합니다.
- **구현 방안:**
    - **Repository 생성:** `PostRepository` 인터페이스를 생성하고 `JpaRepository`를 상속받습니다.
    - **Service 생성:** `PostService` 클래스를 생성하여 이미지 파일 저장, 게시물 정보 저장/수정/삭제 등의 비즈니스 로직을 구현합니다.
    - **Controller 생성:** `PostController` 클래스를 생성하여 아래와 같은 RESTful API 엔드포인트를 제공합니다.
        - `POST /api/posts`: 새 게시물 생성 (이미지 파일 업로드 포함)
        - `GET /api/posts`: 모든 게시물 목록 조회
        - `GET /api/posts?tag={tag}`: 특정 해시태그를 포함한 게시물 목록 조회
        - `GET /api/posts/{id}`: 특정 게시물 상세 조회
        - `PUT /api/posts/{id}`: 특정 게시물 정보 수정
        - `DELETE /api/posts/{id}`: 특정 게시물 삭제

### 5. 3단계: 프론트엔드 구조 설정
- **요구사항:** (SRS: FR-8) `TEXT/index.html`이 초기 페이지로 설정되어야 합니다.
- **구현 방안:**
    - 최상위 경로에 `TEXT` 디렉토리를 생성합니다.
    - `TEXT` 디렉토리 내에 `index.html` 파일과 `css`, `js`, `images` 하위 폴더를 생성합니다.
    - `index.html`에 기본적인 HTML 구조(헤더, 메인, 푸터)를 작성하고, `css/style.css`와 `js/app.js` 파일을 연결합니다.
    - `live-server` 실행 시 루트 디렉토리를 `./TEXT`로 지정하여 `index.html`이 기본으로 표시되도록 합니다.

### 6. 4단계: 프론트엔드 뷰 개발
- **요구사항:** (SRS: FR-3, FR-4, FR-7), (PRD: Awwwards 디자인 참고) 갤러리 그리드, 상세 뷰, 해시태그 필터링 UI가 필요합니다.
- **구현 방안:**
    - **갤러리 뷰 (`index.html`):**
        - `js/app.js`에서 `fetch` API를 사용해 `GET /api/posts`를 호출합니다.
        - 받아온 데이터를 기반으로 썸네일 이미지가 포함된 격자(Grid) 레이아웃을 동적으로 생성합니다.
        - CSS Grid 또는 Flexbox를 사용하여 반응형 레이아웃을 구현합니다.
    - **상세 뷰 (동적 생성):**
        - 갤러리에서 썸네일을 클릭하면, 모달(Modal) 창 또는 별도 페이지로 상세 정보를 표시합니다.
        - `GET /api/posts/{id}`를 호출하여 원본 이미지, 설명글, 해시태그 목록을 표시합니다.
    - **해시태그 필터링:**
        - 해시태그 클릭 시 `js/app.js`에서 `GET /api/posts?tag={tag}` API를 호출하고, 반환된 결과로 갤러리 뷰를 다시 렌더링합니다.

### 7. 5단계: 기능 연결 및 통합 테스트
- **요구사항:** 모든 기능이 유기적으로 동작해야 합니다.
- **구현 방안:**
    - 게시물 작성 폼과 생성(POST) API를 연동하고 테스트합니다.
    - UI에서 게시물 수정 및 삭제 기능을 구현하고 API와 연동하여 테스트합니다.
    - 이미지 업로드 및 표시가 정상적으로 이루어지는지 확인합니다.
    - (NFR-3) Chrome, Firefox 등 주요 브라우저에서 기능 및 레이아웃 호환성을 테스트합니다.

### 8. 향후 계획
- PRD의 '초기 버전 제외 범위'에 명시된 기능들을 점진적으로 추가하는 것을 고려합니다.
    - 사용자 계정 및 인증 시스템 도입
    - 댓글 및 '좋아요' 기능 추가
    - 고급 검색 기능 구현
